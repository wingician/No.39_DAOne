# 贡献指南 - 翼术师世界

## 欢迎加入创作！

感谢你对翼术师世界的兴趣！这是一个开放式协作的魔法世界创作项目，我们欢迎所有形式的贡献。

## 📚 你可以贡献什么？

### 世界设定
- 地理环境：大陆、岛屿、城市、特殊地貌
- 社会人文：节日、习俗、文化传统
- 历史事件：重要战役、文明变迁
- 魔法体系：新的魔法理论、魔法现象
- 生物种族：动物、植物、魔物、智慧种族

### 人物角色
- 角色卡片：背景、性格、能力、关系
- 术师资料：等级、专精、成就

### 故事创作
- 短篇故事：基于世界设定的独立故事
- 长篇小说：系列创作
- 人物传记：角色的完整故事

## 🚀 如何开始

### 1. Fork 仓库

点击右上角的 Fork 按钮，将仓库复制到你的账户下。

```bash
git clone https://github.com/你的用户名/doc.git
cd doc
```

### 2. 创建分支

为你的贡献创建一个新分支：

```bash
git checkout -b feature/你的功能名称
# 例如：
# git checkout -b feature/add-shadowmoon-city
# git checkout -b story/the-tale-of-wing-master
```

### 3. 添加内容

按照文件结构组织你的内容：

```
故事背景/
├── 天文地理和历史/        # 地理相关设定
├── 魔法/                   # 魔法体系
├── 社会人文/               # 文化习俗
├── 时间线(历史)/           # 历史事件
├── 王国和学院/             # 政治机构
└── 翼术师/                 # 术师体系

人物卡片/                   # 角色资料

小故事/                     # 短篇故事创作

小片段/                     # 故事片段
```

### 4. 遵循格式规范

所有文档使用 **Markdown** 格式。参考现有文件的格式：

```markdown
# 标题

## 基本信息
- **位置**：所在大陆/国家
- **类型**：城市/地区/事件等
- **时代**：所属纪元

## 描述

详细描述内容...

## 相关设定

- [[关联设定1]]
- [[关联设定2]]

## 参考资料

如果有参考来源，请注明。
```

### 5. 提交更改

```bash
git add .
git commit -m "描述你的更改"
git push origin feature/你的功能名称
```

### 6. 创建 Pull Request

1. 在 GitHub 上打开你的仓库
2. 点击 "Pull Request" 按钮
3. 填写 PR 描述，说明你的贡献内容
4. 等待自动检查和审核

## 🤖 自动冲突检查

我们的 AI 助手会自动检查你的提交，确保内容与现有设定不冲突。检查项包括：

### 时间线冲突
- 事件发生时间是否符合历史设定
- 角色出现时间是否合理（是否已死亡/未出生）

### 地理冲突
- 地点是否在合理的位置
- 是否与已有地理设定矛盾

### 魔法体系冲突
- 魔法能力是否符合魔法理论
- 魔法强度是否合理
- 是否引入了不符合世界观的元素（如现代科技）

### 人物冲突
- 角色关系是否合理
- 能力等级是否符合设定

### 文化冲突
- 社会习俗是否符合世界设定
- 是否与已有文化传统矛盾

## ✅ 内容规范

### 世界观一致性

**禁止的内容：**
- ❌ 现代科技（电脑、手机、互联网等），除非在特定平行宇宙分支
- ❌ 核武器、现代武器（与世界观不符）
- ❌ 与已有魔法理论严重冲突的设定
- ❌ 打破时间线连贯性的内容

**鼓励的内容：**
- ✅ 基于东方哲学的魔法理论
- ✅ 融合易经、五行、八卦的设定
- ✅ 富有想象力的地理环境
- ✅ 深入的人物刻画
- ✅ 引人入胜的故事情节

### 命名规范

- **地名**：可以使用中文或带有东方韵味的名称
  - 例如：稻一大陆、九神州、凄海、锡金、柳林镇
- **人名**：可以使用西方化的名字，但内涵是东方的
  - 例如：阿尔锋、麦真、欧内斯特
- **术语**：保持与现有术语一致
  - 翼术师、法术师、元神、魔地等

### 质量标准

- **原创性**：内容必须是你的原创，或有合法授权
- **完整性**：设定应该完整，不要留下过多未解释的内容
- **可读性**：语言流畅，格式清晰
- **关联性**：与现有世界观有良好的衔接

## 🌳 分支管理

### 主分支 (main)
主世界线的正史设定。要求最严格，需要经过详细审核。

### 平行宇宙分支

你可以创建新的分支探索不同的世界线：

```bash
# 创建蒸汽朋克分支
git checkout -b branch-steampunk

# 创建黑暗中世纪分支
git checkout -b branch-darkage

# 创建赛博魔法分支
git checkout -b branch-cybermagic
```

平行宇宙分支的规则可以更灵活，但需要在分支说明中清楚标注与主线的差异。

## 📝 PR 模板

提交 Pull Request 时，请使用以下模板：

```markdown
## 贡献类型
- [ ] 世界设定
- [ ] 人物角色
- [ ] 故事创作
- [ ] 修复错误
- [ ] 其他

## 描述
简要描述你的贡献内容

## 相关设定
列出与你的内容相关的已有设定

## 检查清单
- [ ] 内容符合世界观
- [ ] 没有时间线冲突
- [ ] 没有地理冲突
- [ ] 格式规范正确
- [ ] 已阅读贡献指南
```

## 🎨 文档建议

### 使用内部链接

使用 `[[链接]]` 语法链接到其他设定文档：

```markdown
这个魔法理论基于[[混沌能量]]，在[[魔法纪元]]达到顶峰。
```

### 添加标签

为你的文档添加相关标签：

```markdown
#地理 #南烛大陆 #魔法学院
```

### 引用来源

如果参考了易经、道家思想等，请注明：

```markdown
> 此魔法理论基于《易经》的阴阳变化思想
```
